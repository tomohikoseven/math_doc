# Antigravity IDE Agents Rules

このプロジェクト（ワークスペース）全体に適用される共通ルールです。

- **Gitコミット**: コミットを行う際のコミットメッセージは、必ず**日本語**で記述してください。
- **段落の分離**: 文章の段落（全角スペース「　」から始まる新しい段落など）が分かれる場合は、Markdownのレンダリング上で正しく段落を分離させるため、**段落間に必ず1行の空行を挿入**してください。改行のみで次の段落を始めることは避けてください。
- **太字記法**: Markdownの太字記号（`**`）は一切使用せず、**常にHTMLの `<b>` タグを使用**してください（例: `<b>太字の内容</b>`、`<b>「括弧付きの内容」</b>`）。
- **セクション区切り**: 見出し（セクション）の境界に水平線（`---`）を挿入しないでください。
- **デザイン（スタイル）構築**: 独自コンポーネントを作成・修正する際のデザイン（CSS）は、可能な限り生の `<style>` タグではなく、**Tailwind CSS (v4系)** のユーティリティクラスを優先して使用してください。Astro Starlight環境におけるダークモード対応（`dark:` バリアント）やカスタムプロパティ（`var(--sl-color-...)`）の活用も含め、インラインクラスでのスタイリングを行います。
  - **長いTailwindクラスのコンポーネント化**: `[&_ul]:list-none` のような任意の子要素セレクタ（arbitrary variants）などでTailwindクラスが非常に長くなる場合、MDXファイル等へ複数回直書きしてコードを肥大化させるのは避けてください。対象の要素をAstroコンポーネントとして切り出すか、グローバルCSS（`@layer components` 内での `@apply` 活用）にクラスとして定義することで、MDX側の可読性と保守性を保つようにしてください。
- **Markdownの表**: Markdown表が3列以上になる場合は、2列になるようにするか、別の表現方法を検討してください。
- **禁止用語**: 数学的な記事において、「宇宙」という言葉の使用は禁止します。比喩として用いる場合でも誤解を招くため、「対象となる世界」「議論の前提となる領域」「複雑な現象全体」「対象とする体系」など、文脈に応じた別の代替用語や具体的な説明を使用してください。
- **AIエージェント向け Open Knowledge Format (OKF) 対応**:
  - 本リポジトリのドキュメント（MD/MDXファイル）は、人間とAIエージェントの双方が理解しやすいよう、**Google CloudのOpen Knowledge Format (OKF)** の要件に準拠させています。
  - AIアシスタントが新規ファイルを作成する、または既存ファイルを大幅に修正する場合は、必ずフロントマター（frontmatter）に以下のメタデータを含めてください。
    - `type` (必須): そのドキュメントの性質を示す文字列（例: `"concept"`, `"runbook"`, `"system"`, `"faq"` など）。
    - `description` (必須): ドキュメントの簡潔な要約。AIエージェントが検索やインデックス化する際に有用な情報を100文字程度で記載。
    - `tags` (任意): 関連するキーワードの配列。
    - `resource` (任意): 関連する外部URLやシステムへのリンクがある場合に指定。
  - これにより、本リポジトリ全体がそのままAIエージェント用の「ナレッジベース」として最適化されます。
