# Astroコンポーネントの使用ルールとAEO対策

- **Markerコンポーネントの使い分け**: 強調表現のうち、文章中の重要なフレーズには `<Marker>...</Marker>` を使用してください。特に、各見出し（セクション）単位で要点や最も重要な箇所を1つ選び、必ず `<Marker>` タグで囲んで強調するようにしてください。ただし、リスト内の先頭（見出し部分）など、項目のタイトルとしての強調には `<Marker>` ではなく `<b>` を使用してください。
- **数式と太字の併用時の注意**: `<b>` タグの範囲内にインライン数式（`$`）を含めることは可能ですが、MDXのパースエラーを防ぐため、可能な限り数式は太字の外に配置することを推奨します。Markdownの `**` は使用禁止です。
- **AEO (Answer Engine Optimization) 対策**: AI 回答エンジン（Perplexity, SearchGPT 等）が情報を正確に抽出できるよう、以下の構成を徹底してください。
  - **冒頭での直接回答**: 記事の冒頭（導入部分）で、タイトルの問いに対する明確な結論や定義を簡潔に提示してください。
  - **パラグラフ・ライティングの徹底**: 記事全体を「総論→各論→まとめ」の構成とし、各見出し（H2, H3）の直下には必ず「そのセクションの要約（結論）」を配置してから「詳細（各論）」を展開してください。ただし、「総論」「要約」「各論」という単語を直接見出し（`## 総論`など）として使用しないでください。
  - **Q&A 形式の活用**: 読者が検索しそうな具体的な疑問を小見出しや専用セクション（`FAQ` 等）として作成し、それに対する直接的な回答を記述してください。
  - **構造化コンポーネントの活用**: `Aside`, `Card`, `Steps` 等の標準コンポーネントを使用して、重要なポイントやプロセスを視覚的・構造的に際立たせてください。ただし、`Aside` コンポーネントなどを直接使用する前に、`/src/components/` 内の独自コンポーネント（`Statement.astro` や `ArticleTip.astro` など）で要件を満たせないか確認し、独自コンポーネントの使用を優先してください。FAQセクションを作成する場合は、必ず独自コンポーネントの `FAQ.astro` と `Statements.astro`（`<Statements><FAQ title="...">...</FAQ></Statements>`）を使用してください。また、単純なリスト（箇条書きや番号付きリスト）を作成する場合は、必ず独自コンポーネントの `FlatList.astro`（`<FlatList>...</FlatList>`）でリスト全体を囲んでください。
  - **Schema.orgの追加**: FAQセクションを作成・修正した場合は、必ずMDXのフロントマター（frontmatter）の `schema:` プロパティに `schema.org` の `FAQPage` 構造化データを追加・更新してください（`@type: FAQPage` として `mainEntity` に各Q&Aのペアを記述します）。
  - **Stepsコンポーネントの書き方**: **番号リストだからといって必ずしも `<Steps>` コンポーネントを使うわけではありません。** `<Steps>` コンポーネントは、内容が「作業などの手順」を示す場合にのみ使用してください。手順ではない単なる項目の列挙（構成要素のリストや概念の対応関係など）で番号付きリストを使う場合は、`<Steps>` ではなく必ず `<FlatList>` を使用してください。また、`<Steps>`内で番号付きリスト（例: `1. <b>見出し</b>`）を使用する場合、見出しとそれに続く説明文の間には必ず1行の空行を挿入してください。空行がないとレンダリング時に見出しと文章が分離されず視認性が低下します。
  - **リスト（箇条書き）の記号**: 順序を持たない箇条書きを作成する際は、アスタリスク（`*`）ではなくハイフン（`-`）を箇条書きの記号として使用してください。
  - **太字の見出し表現**: リストのネストを避けてタイトル付きの説明を記述する場合、見出しと説明文を別行に分けるのではなく、`<b>見出し</b>：説明文` のように全角コロン（：）で繋いで1行にまとめるフォーマットを推奨します。これにより、視認性を保ちながら縦幅を節約できます。
  - **簡潔な結論セクション**: 記事の最後には必ず「まとめ」や「結論」のセクションを設け、重要な要点を箇条書きで整理してください。
  - **関連用語の明記禁止**: 記事内の見出しや本文中に「AEO対応」や「AEO対策」といったシステム側の文言は直接記述しないでください。読者にとって自然で洗練された技術解説・ドキュメントであることを優先します。
  - **メタディスクリプション（SEO/GEO対策）**: フロントマターの `description` および `schema` 内の `description` は、検索エンジンやAI回答エンジンで適切に表示・抽出されるよう、**日本語で100〜140文字程度**の具体的かつ詳細な内容を記述してください。短すぎる説明文（Webmaster Toolsの警告対象）は避けてください。
  - **執筆スタイル**: 数学的・技術的な解説文では、一貫して「である調」（常体）を使用してください。「です・ます調」（敬体）は使用しないでください。
