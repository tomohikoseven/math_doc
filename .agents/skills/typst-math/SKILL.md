# 数式の記述ルール (Typst)

このプロジェクトでは数式の記述に **Typst** を使用します。以下の指示に従ってください。

- **数式記法**: 数式は必ず Typst の記法（`$ ... $`）を使用してください。**LaTeX 記法（\ から始まるコマンド）は一切使用しないでください。**
  - **Typst ではバックスラッシュ（\）を使いません**。
  - ギリシャ文字: `alpha`, `beta`, `epsilon`, `phi`（バックスラッシュなし）
  - 演算子: `times` (×), `dot` (・), `divide` (÷)
  - 省略記号: `dots.h` (...)
  - その他: `sqrt(x)`, `sum`, `integral`, `infinity`
  - **無限大の記号**: `oo` は未定義変数エラーを引き起こしてMDXのビルドを破壊するため絶対に使用せず、必ず `infinity` または `infin` を使用してください。
- **変数と記号の結合防止**: 変数とコマンド、あるいは変数どうしが結合して未定義変数（unknown variable）にならないよう、適宜スペースを入れてください。
  - 悪い例: `$ax = b$`, `$dx$`
  - 良い例: `$a x = b$`, `$d x$`
- **数学記号**: Typst の標準的なシンボルを使用してください。
  - Not equal: `!=` または `neq`
  - 矢印: `->`, `=>`, `==>` など
  - 分数: `num / den` または `frac(num, den)`。インライン数式（`$`）内で分数を使用する場合は、小さく潰れるのを防ぐため必ず `display()` をつけてください（例: `$display(num / den)$`、`$display(x = 1/2)$`）。数式ブロック（`$$`）内は不要です。
- **数式ブロック**: 独立した数式行にする場合は、MDX のパースエラー（特に `{` などの記号）を避けるため、必ず **`$$ ... $$`** 形式を使用してください。
- **数式ブロック（$$）の安定性**: `$$` ブロック内に日本語の文字列（`"..."`）を混在させるとレンダリングが不安定になる場合があります。極力、論理記号（`and`, `or` 等）を使用するか、日本語の説明は数式ブロックの外（前後のテキスト）に記述してください。

# 図表描画 (Fletcher) 構文ルール

- インポートパスは必ず以下のように絶対パスで記述すること:
  `#import "/libs/fletcher/src/exports.typ" as fletcher: diagram, node, edge`
- ノードのラベルテキストは、必ずコンテンツブロック `[...]` で囲むこと (例: `node((0, 0), [ラベル])`)。
- ノードラベル内での改行は、バックスラッシュ `\` を使用すること (例: `[微分 \ =分数の極限]`)。
- `edge` の接続、矢印タイプ（`"-"`, `"-|>"`）およびオプション引数（`label: [...]`, `label-side: left` 等）が正しいか厳密に確認すること。
- 各引数の区切り（カンマ `,`）や全体の括弧のペア（`(` と `)`）が閉じられているか、必ず構文チェックすること。
- Fletcherを使用しない数式ブロックでは、不要な `fletcher` の `#import` を記述しないでください。

# 図形・レイアウト関数（#box, #grid 等）の記述ルール

- **バージョン制約**: 現在の環境（Typst 0.10.0 相当）では、`#grid` 関数などに `align` 引数を指定すると「unexpected argument」エラーとなり表示されません。配置の調整は必ず `#align(center)[ ... ]` で全体を囲むか、コンテナの直下で `#set align(center)` を使用してください。
- **数式ブロック（`$$`）内の幅指定**: `$$` ブロックは中身に合わせて自動で幅が縮小（シュリンクラップ）されます。そのため、内部の `#box` 等に `width: 100%` などの相対指定をすると幅が 0pt に計算され、図形が消えてしまいます。図形のサイズを決める際は必ず `width: 400pt` などの**固定幅（絶対指定）**を使用してください。
