スマートフォン専用のWebサイトや強固なアクセス制限を持つポイントサイト（ポイ活）を自動化するためには、PCブラウザの偽装ではなく、Androidの標準デバッグ機能である「**ADB (Android Debug Bridge)**」とNode.jsを用いた物理端末の直接制御が最も確実である。本記事では、外部の複雑なブラウザ自動化ライブラリを使わず、ADBコマンドのみでAndroid上のChromeを操作する基礎的な手法を解説する。

## はじめに

現代の「ポイ活（ポイント活動）」の中には、スマートフォン端末からアクセスしないと実行できない案件が数多く存在する。PCのブラウザでユーザーエージェントを偽装しても弾かれてしまうような強固なサイトに対抗するためには、本物のAndroid端末をPCから物理的・システム的に操作（自動化）するアプローチが最も確実である。

本記事では、PlaywrightやVibiumといった高度なブラウザ自動化ライブラリを使わず、Androidの標準デバッグ機能である **ADB (Android Debug Bridge)** と **Node.js** だけを用いて、極限まで軽量かつ確実にAndroid上のChromeを自動起動し、指定のWebサイト（今回はGoogle）を開くまでの基礎を解説する。

## 環境情報

本記事の手順は、以下の環境で動作確認を行っている。

- **OS**: Windows 10 Pro
- **Node.js**: v24.11.1
- **ADB**: Android Debug Bridge version 1.0.41
- **端末**: Android OS 13 スマートフォン (Chromeブラウザインストール済み)

<Aside type="note" title="なぜ高度なライブラリを使わないのか？">
PlaywrightのAndroidサポートは非常に強力であるが、端末やセキュリティ設定によっては「高度なプログラム制御（CDP接続）」がブロックされ、真っ白な画面から進まないエラーが発生することがある。手元のZTE Libero 5G Ⅲでも同様の事象が確認された。ADBの標準コマンドのみを利用する「純粋ADB制御」であれば、そうした制約をすり抜けて、人間が手で操作しているのと全く同じレベルの自動化が実現可能である。
</Aside>

## セットアップ方法

自動化を始める前に、PCとスマートフォン双方の準備が必要となる。

### 1. PC側の準備

<Steps>
1. **Node.js のインストール**
   [Node.js公式サイト](https://nodejs.org/) から最新版をインストールする。
1. **ADBコマンドの導入**
   [Android SDK Platform-Tools](https://developer.android.com/studio/releases/platform-tools) をダウンロードし、解凍したフォルダへのパスをWindowsの「環境変数 (Path)」に登録する。
   ターミナル（PowerShellなど）で `adb version` と入力し、バージョン情報が表示されれば成功である。
</Steps>

### 2. スマートフォン側の準備

<Steps>
1. **開発者向けオプションの解放**
   設定の「端末情報」などにある「ビルド番号」を7回連続でタップし、開発者モードを有効にする。
1. **USBデバッグの有効化**
   「開発者向けオプション」メニューに入り、「USBデバッグ」をONにする。
1. **PCとの接続と権限許可**
   USBケーブルでPCとスマートフォンを繋ぐ。
   - スマホ画面に「USBデバッグを許可しますか？」と出たら **許可（OK）** を押す。
   - USBの接続用途を聞かれたら **「ファイル転送（MTP）」** を選択する。
1. **画面ロックの解除**
   自動化ツールを動かす際は、**必ず画面ロックを解除し、画面をオン（アクティブ状態）**にしておく必要がある。画面が消灯していると操作コマンドが反映されない。
</Steps>

### 3. 接続テスト

コマンドプロンプトやPowerShellを開き、以下のコマンドで接続を確認する。

```powershell
adb devices
```

```text title="実行結果のサンプル"
List of devices attached
1234567890abcdef    device
```

`device` と表示されていれば完了である。もし `unauthorized` や `offline` と表示された場合は、スマホの画面に許可ダイアログが出ていないか確認し、ケーブルを挿し直すこと。

## ADBコマンド単体でChromeを起動してみる

Node.jsで本格的なスクリプトを書く前に、まずはADBコマンドだけで直接スマホを操作できることを確認する。
ターミナル（PowerShell等）に以下の1行をそのまま貼り付けて実行する。

```powershell
adb shell am start -a android.intent.action.VIEW -d "https://www.google.com" com.android.chrome
```

このコマンドを実行した瞬間、スマートフォン上でChromeが立ち上がり、Googleのトップページが表示されれば成功である。
このように、ADBを使えばコマンド一つでアプリの起動やURLの指定が可能となる。これをNode.jsから連続して実行させるのが、今回の自動化の仕組みである。

## 実装：Node.jsからADBを使って自動化する

準備が整ったら、実際にNode.jsからADBコマンドを発行してスマホを制御する。
外部ライブラリ（npmパッケージ）のインストールは一切不要である。標準モジュールの `child_process` を使用する。

適当なフォルダに `launch_adb.js` というファイルを作成し、以下のコードを記述する。

```javascript title="launch_adb.js"
const { execSync } = require('child_process');

// ADBコマンドを簡単に実行するためのヘルパー関数
function adbShell(command) {
    try {
        console.log(`[実行] adb shell ${command}`);
        // stdioの設定で、余計なエラー出力を抑制しつつ実行
        return execSync(`adb shell ${command}`, {
            encoding: 'utf-8',
            stdio: ['ignore', 'pipe', 'ignore']
        });
    } catch (error) {
        console.error("コマンドの実行に失敗しました。スマホが接続されているか確認してください。");
        process.exit(1);
    }
}

// 待機用の便利関数（ミリ秒指定）
const sleep = (ms) => new Promise(resolve => setTimeout(resolve, ms));

async function main() {
    console.log('===== ADB自動操作テスト =====');

    // 1. 指定のURL（Google）をChromeで開くインテント（命令）を送信
    const url = "https://www.google.com";
    const pkg = "com.android.chrome"; 
    
    console.log('Chromeを起動してGoogleを開きます...');
    adbShell(`am start -a android.intent.action.VIEW -d "${url}" ${pkg}`);

    // ブラウザが起動し、ページが読み込まれるのを待つ
    await sleep(5000); 

    // 2. 例として、画面の少し下をタップする操作
    // ※ 座標(x=500, y=1000) はお使いの端末に合わせて変更すること
    console.log('画面をタップします...');
    adbShell('input tap 500 1000');

    console.log('\n動作確認完了です！スマホの画面を見てみましょう。');
}

main();
```

### 実行方法

ターミナルで先ほど作成したファイルの場所に移動し、Node.jsで実行する。

```powershell
node launch_adb.js
```

スマホの画面が自動で切り替わり、Chromeが立ち上がってGoogleのトップページが表示されれば成功である。

<Aside type="tip" title="補足：今回使用したADBコマンドの解説">
スクリプト内で使用している、Android特有のコマンドの動作を解説する。

1. **`am start -a android.intent.action.VIEW -d "URL" パッケージ名`**
   Androidの「Activity Manager (`am`)」に対して、指定したURLを開くActivity（画面）を起動せよ、という命令（インテント）を送るコマンドである。
   末尾の `com.android.chrome` を別のブラウザのパッケージ名（例：Braveなら `com.brave.browser`）に変更すれば、任意のブラウザでサイトを開くことができる。

2. **`input tap X Y`**
   画面上の指定した座標 (X, Y) を1回タップする。自動化において最も直感的で多用するコマンドである。
</Aside>

<Aside type="note" title="より高度な自動化のためのADBコマンド">
本格的なポイ活スクリプトを作る際は、以下のコマンドも非常に有用である。

1. **`input swipe X1 Y1 X2 Y2 [期間ms]`**
   指定した座標（X1, Y1）から（X2, Y2）へとスワイプ（スクロール）する。画面を下にスクロールして「続きを読む」ボタンを探す際などに必須となる操作である。期間を指定することで、「ゆっくりスワイプする」「素早く弾く（フリック）」を切り替えられる。

2. **`uiautomator dump /dev/tty`**
   現在画面に表示されている要素（ボタンのテキストや構造などのHTMLに近いもの）をXML形式で一気に取得する。
   例えば、XMLの中から「スタンプGET」という文字列のX・Y座標をNode.js側で計算し、そこに向かって `input tap` を実行することで、端末ごとの画面サイズの違いに影響されないスマートな自動タップが実現可能になる。
</Aside>

## まとめ

* PCブラウザの偽装が通じないサイトの自動化には、Android端末を直接操作するADBコマンド制御が有効である。
* Node.jsの `child_process` を用いることで、外部ライブラリに依存せず、軽量かつ確実にスマホのChromeを起動・操作できる。
* `input tap` や `input swipe`、`uiautomator dump` を組み合わせることで、複雑な操作を全自動化する独自のBotを構築可能である。
