<ArticleTip title="結論" wid="800">

`astro.config.mjs` に `site` オプション（公開URL）を追加するだけで、ビルド時に自動的にサイトマップ（`sitemap-index.xml`）が生成される。

</ArticleTip>

Astroでビルドを実行した際、ターミナルに以下のような警告が出力され、サイトマップが生成されない場合がある。

<Aside type="danger" title="出力される警告">
```sh
[WARN] [@astrojs/sitemap] The Sitemap integration requires the `site` astro.config option. Skipping.
```
</Aside>

この警告は、Cloudflare Pagesなどの環境でAstroをビルドしたときにも見られる。原因は `astro.config.mjs` に `site` オプションが設定されていないため、サイトマップの基準URLが特定できず、生成処理がスキップされていることである。

## 解決策：astro.config.mjsにsiteを追加する

警告文が示す通り、`astro.config.mjs` にサイトの基本URLを指定する `site` プロパティを追加すれば解決する。

<Steps>

1. `astro.config.mjs` を開く。

1. `defineConfig` 内に `site: 'https://あなたのドメイン.com'` を追記する。

   ```js {3,3} title="astro.config.mjs"
   // astro.config.mjs
   export default defineConfig({
     site: 'https://xxx.example.com',
     integrations: [...],
     // ...
   });
   ```

1. 設定を保存し、再度ビルドを実行する。

</Steps>

これにより、ビルド時に `public` または `dist` ディレクトリへ `sitemap-index.xml` が正常に作成される（例: [sitemap-index.xml](https://mathdoc.ifdef.jp/sitemap-index.xml)）。

<Aside type="note" title="ライブラリの追加は不要">
インターネット上の記事には「`@astrojs/sitemap` を手動でインストールする」よう促すものがあるが、Starlightなどを利用している場合は標準で組み込まれていることが多いため、追加のインストール作業は不要である。`site` を追加するだけで自動生成機能が有効になる。
</Aside>

## まとめ

* Astroビルド時のサイトマップ関連警告は、`astro.config.mjs` での `site` 指定漏れが原因である。
* `defineConfig` 内に自身のWebサイトのURLを `site: 'https://...'` として追記することで警告は消え、サイトマップが生成される。
