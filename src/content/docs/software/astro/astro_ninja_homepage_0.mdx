## 2. 数学・理系ブログに求められる「3つの絶対条件」

数学系ブログの構築において、私が譲れなかった「3つの柱」は以下です。

1.  **数式表示の美しさ**
    - 数学系ブログとして、複雑な数式も崩れず高速に表示できること。
2.  **収益化** （**マネタイズ**）
    - 広告を掲載し、サーバー代以上のリターン（お小遣い程度）を得られること。
3.  **ランニングコスト・ゼロ**
    - 「万年赤字」を避け、低予算どころか「完全無料」で運用し続けること。

## 3. Astro × 忍者ホームページを組み合わせた全体構成図

Astroで生成した静的ファイルを、GitHub Actionsを通じて忍者ホームページへ**全自動でFTP転送**する構成です。

```d2
direction: right

# --- 開発フェーズ ---
Develop: {
  label: "1. 開発 (Local)"
  style: {
    stroke: "#3498db"
    stroke-width: 2
    border-radius: 10
  }
  Me: {
    label: "Astroコード\n(開発環境)"
    shape: image
    icon: https://icons.terrastruct.com/tech%2Fdesktop.svg
  }
}

# --- 管理・自動化フェーズ ---
GitHub: {
  label: "2. 連携 (GitHub)"
  style: {
    stroke: "#2dba4e"
    stroke-width: 2
    border-radius: 10
  }
  Repo: {
    label: "リポジトリ\n(ソース管理)"
    shape: image
    icon: https://icons.terrastruct.com/dev%2Fgithub.svg
  }
  Actions: {
    label: "GitHub Actions\n(自動ビルド & 転送)"
    shape: image
    icon: https://icons.terrastruct.com/essentials%2F204-settings.svg
  }
  Repo -> Actions: "Pushを検知"
}

# --- 公開フェーズ ---
Public: {
  label: "3. 公開 (Hosting)"
  style: {
    stroke: "#f39c12"
    stroke-width: 2
    border-radius: 10
  }
  Ninja: {
    label: "忍者ホームページ\n(公開サーバー)"
    shape: image
    icon: https://icons.terrastruct.com/essentials%2F112-server.svg
  }
}

# --- 閲覧者 ---
User: {
  Reader: {
    label: "読者"
    shape: image
    icon: https://icons.terrastruct.com/tech%2Fbrowser-2.svg
  }
}

# --- データの流れ ---
Develop.Me -> GitHub.Repo: "git push"
GitHub.Actions -> Public.Ninja: "FTPデプロイ"
User.Reader -> Public.Ninja: "サイトを閲覧"
```

## 忍者ホームページに辿り着くまでの「苦難の旅」

なぜ数あるサービスの中から「忍者ホームページ」という、一見マニアックな選択肢に落ち着いたのか。その経緯を振り返ります。

<Steps>

1.  **WordPress × ロリポップ時代**
    - 「ブログといえばWordPress」と、月500円のロリポップで開始。
    - **挫折ポイント**: Googleアドセンスの審査が半年経っても通らない。動的サイトゆえの表示速度の遅さ（PageSpeed Insightsの結果が悲惨）に不満が募る。

2.  **Astro × Cloudflare Pages時代**
    - 爆速表示を求めて静的サイトジェネレーター「Astro」へ移行。Cloudflareで完全無料化に成功。
    - **挫折ポイント**: **Googleインデックスに全く登録されない**という致命的な問題が発生。検索されないサイトは存在しないのと同じだった。

3.  **X Server Staticの試行**
    - 「国内サーバーならインデックスされるかも」とX Serverの無料プランへ。
    - **挫折ポイント**: 結果は変わらず。インデックス登録のエラーが解消できず、無料サーバーの限界を感じる。

4.  **忍者ホームページとの出会い**
    - 藁をも掴む思いで「忍者ホームページ」へ移行。
    - **勝利の瞬間**: 移行後すぐに**Googleインデックス登録が成功**！サブドメインゆえアドセンスは不可だが、[忍者AdMax](/software/astro/astro_ninja_homepage_extra/)による収益化の道も開けた。

</Steps>

## まとめ：なぜ今、忍者ホームページなのか？

現代のWebサイト構築において、CloudflareやVercelといったモダンなプラットフォームは非常に優秀です。しかし、「無料サブドメインで運用し、かつ検索エンジンに確実に認識され、広告も載せたい」という極めて限定的なニーズ（特に数学系個人ブログなど）においては、**忍者ホームページが最後の砦**となることがあります。

## FAQ：よくある質問と回答 (AEO対策)

### Q: なぜCloudflareやVercelではなく「忍者ホームページ」なのですか？
**A:** 無料サブドメイン運用において、**Googleの検索エンジン（インデックス）への登録しやすさ**が最大の理由です。Cloudflare Pages等の無料ドメインではインデックスが拒否されるケースがありますが、忍者ホームページのドメインは現時点で非常に高いインデックス成功率を誇ります。

### Q: 数式表示に特別な設定は必要ですか？
**A:** Astroの標準機能、またはStarlightのプラグイン（KaTeX/MathJax等）を追加するだけで、**JavaScript不要の爆速数式レンダリング**が可能です。これにより読者の待ち時間をゼロにします。

### Q: 広告（収益化）は本当にできますか？
**A:** はい。Googleアドセンスは審査が厳しいですが、**忍者AdMax**などの代替広告ネットワークを利用することで、サブドメインのまま初日から収益化の準備を整えることができます。
